Option+SStandardkVK_ANSI_S (1).maskAlternate
Option+MSocial MediakVK_ANSI_M (46).maskAlternate

Detection: CGEventTap at cgSessionEventTap with headInsertEventTap. Events are swallowed (return nil) to prevent characters reaching the active app.

Modifier filtering: Only Option key. Command and Control are rejected to avoid conflicts.

Release detection: Key up stops recording. Option release via flagsChanged also stops as safety fallback.

Claude System Prompts

Standard Mode (Option+S)

You are a text polishing assistant. The user has dictated a message via voice. Your task is to clean up the transcription: fix grammar, improve flow and readability, remove filler words and repetitions. Keep the original meaning and language (German or English, match the input). Keep the same tone and register. Output ONLY the improved text, nothing else.

Social Media Mode (Option+M)

You are a text polishing assistant. The user has dictated a message via voice. Clean up the transcription: fix grammar, improve flow and readability, remove filler words and repetitions. Add exactly [N] emojis placed naturally throughout the text. IMPORTANT: Do NOT add any new content, ideas, sentences or information that the user did not say. Only clean up what was spoken and add emojis. Keep the original meaning, length and language (German or English, match the input). Output ONLY the improved text, nothing else.

The emoji count [N] is configurable in Settings (default: 3, range: 1–20).

Accessibility During Development

When running from Xcode, the built binary in DerivedData needs Accessibility permission. This resets after each rebuild (code signature changes). Add the app from ~/Library/Developer/Xcode/DerivedData/SpeechToText-.../Build/Products/Debug/SpeechToText.app to System Settings → Accessibility. This is not an issue with exported/archived builds.

End-to-End Pipeline

 1. User holds Option+S or Option+M
 2. HotkeyManager detects keyDown + maskAlternate
 3. SpeechCoordinator.didStartRecording(mode:)
 4. PasteManager saves frontmost app
 5. AudioRecorder starts M4A recording (16kHz mono)
 6. Menu bar icon → red mic
 7. User releases key
 8. HotkeyManager detects keyUp → didStopRecording
 9. AudioRecorder stops, returns temp file URL
10. File size check: < 1KB → silently discard
11. Menu bar → "..." (Whisper processing)
12. WhisperService transcribes via OpenAI API
13. Menu bar → sparkles (Claude processing)
14. ClaudeService enhances text
15. PasteManager re-activates saved app + Cmd+V
16. Menu bar → idle state
17. AudioRecorder.cleanup() removes temp file

How to Build & Run

git clone https://github.com/MINECUMN/SpeechToText.git
cd SpeechToText
open SpeechToText.xcodeproj
# In Xcode: Cmd+R

Command line build:

xcodebuild -project SpeechToText.xcodeproj -scheme SpeechToText -configuration Debug build

Export for Another Mac

  1. Open project in Xcode
  2. Product → Archive
  3. Distribute App → Copy App
  4. Copy SpeechToText.app to target Mac's /Applications/
  5. On first launch: grant Microphone + Accessibility
  6. Menu bar icon → Settings → enter API keys
